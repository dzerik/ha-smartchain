## Обзор

**SmartChain** — кастомная интеграция Home Assistant, предоставляющая интеллектуального голосового/текстового ассистента на базе больших языковых моделей (LLM).

Поддерживаемые провайдеры:

| Провайдер | Тип | Особенности |
|-----------|-----|-------------|
| **GigaChat** (Сбер) | Облако | Русскоязычная модель, цензура, vision |
| **YandexGPT** | Облако | Модели Яндекса, суммаризация |
| **OpenAI** | Облако | GPT-4.1, GPT-4o, o3, o4-mini |
| **Ollama** | Локально | Llama, Qwen, Gemma, DeepSeek, Home-3B — без облака |
| **DeepSeek** | Облако | Самая доступная цена за токен |
| **Anthropic** | Облако | Claude Sonnet, Haiku, Opus |

Ключевые возможности:
- Потоковые ответы (токен за токеном)
- Управление устройствами через tool calling
- Несколько агентов с разными моделями и промптами
- Делегирование задач между агентами
- Анализ истории состояний устройств
- Распознавание изображений с камер
- Загружаемые навыки из YAML-файлов
- Сервис для автоматизаций (Telegram, Slack)
- MCP-серверы как дополнительные инструменты
- Кэширование промптов для экономии токенов
- Панель SmartChain AI для генерации и деплоя автоматизаций

## Установка

### Требования

- Home Assistant **2024.12.0+** (для AI Task — 2025.3+)
- [HACS](https://hacs.xyz/) установлен

### Установка через HACS

1. Откройте **HACS** в боковой панели
2. Нажмите меню (три точки) > **Пользовательские репозитории**
3. Введите URL: `https://github.com/dzerik/ha-smartchain`
4. Категория: **Интеграция**
5. Нажмите **Добавить**
6. Найдите **SmartChain** в списке и нажмите **Скачать**
7. Перезапустите Home Assistant

### Ручная установка

1. Скачайте содержимое `custom_components/smartchain/` из [репозитория](https://github.com/dzerik/ha-smartchain)
2. Скопируйте папку `smartchain` в `config/custom_components/`
3. Перезапустите Home Assistant

## Подключение LLM-провайдера

### Шаг 1. Добавление интеграции

**Настройки > Устройства и службы > Добавить интеграцию > SmartChain**

### Шаг 2. Выбор провайдера

На первом экране выберите LLM-провайдер из списка.

### Шаг 3. Ввод учетных данных

| Провайдер | Что нужно | Где получить |
|-----------|-----------|-------------|
| GigaChat | Авторизационные данные (credentials) | [developers.sber.ru/studio](https://developers.sber.ru/studio) |
| YandexGPT | API-ключ + Folder ID | [console.yandex.cloud](https://console.yandex.cloud) |
| OpenAI | API-ключ | [platform.openai.com/api-keys](https://platform.openai.com/account/api-keys) |
| Ollama | Адрес сервера (по умолчанию `http://localhost:11434`) | Установите [Ollama](https://ollama.ai) локально |
| DeepSeek | API-ключ | [platform.deepseek.com](https://platform.deepseek.com) |
| Anthropic | API-ключ | [console.anthropic.com](https://console.anthropic.com) |

> **Совет:** Поставьте галочку **"Пропустить проверку"**, если хотите сохранить конфигурацию без проверки подключения к API.

Можно добавить несколько провайдеров — каждый добавляется как отдельная интеграция.

## Настройка агента

После добавления провайдера перейдите в настройки интеграции: **SmartChain > Настроить**.

### Основные параметры

| Параметр | Описание |
|----------|----------|
| **Модель** | Выберите модель из списка. Список зависит от провайдера |
| **Своё имя модели** | Введите произвольное имя модели (например, `gpt-4.1-2025-04-14`). Если заполнено — используется вместо выбранной из списка |
| **Assist API** | Выберите один или несколько API для управления устройствами (см. [Управление устройствами](#управление-устройствами-assist-api)) |
| **Системный промпт** | Jinja2-шаблон, определяющий поведение ассистента (см. [Системный промпт](#системный-промпт-jinja2)) |
| **Температура** | Креативность ответов. `0.0` — точные, `1.0` — творческие. По умолчанию: `0.1` |
| **Максимум токенов** | Ограничение длины ответа. Оставьте пустым для значения по умолчанию модели |
| **История сообщений** | Включает память диалога (многоходовые разговоры). По умолчанию: включена |
| **Встроенный процессор HA** | Перед отправкой в LLM пробует распознать команду нативным процессором HA. По умолчанию: включен |
| **Инструмент истории состояний** | Позволяет LLM запрашивать прошлые состояния устройств (см. [История состояний](#инструмент-истории-состояний)) |

### Параметры только для GigaChat

| Параметр | Описание |
|----------|----------|
| **Цензура** | Включает фильтр нецензурной лексики. По умолчанию: выключена |
| **Проверка SSL** | Проверка SSL-сертификатов при подключении. По умолчанию: выключена |

### Активация ассистента

1. Перейдите в **Настройки > Голосовые ассистенты**
2. Нажмите **Добавить ассистент**
3. В поле **Conversation agent** выберите вашу сущность SmartChain
4. Сохраните

Теперь ассистент доступен через голосовой пайплайн, чат в боковой панели и автоматизации.

## Несколько агентов (Sub-entries)

SmartChain позволяет создавать несколько независимых агентов на одном провайдере. Каждый агент имеет свою модель, промпт, температуру и набор инструментов.

### Создание агента

1. Перейдите в **Настройки > Устройства и службы > SmartChain**
2. На карточке интеграции нажмите **Добавить агента** (Add conversation agent)
3. Заполните параметры: модель, промпт, температуру и т.д.
4. Нажмите **Подтвердить**

Каждый агент создает отдельную сущность `conversation.smartchain_*`, которую можно выбрать как отдельного голосового ассистента.

### Примеры использования

- **Агент "Умный дом"** — модель GPT-4.1-mini, Assist API включен, промпт про управление устройствами
- **Агент "Повар"** — модель Claude Sonnet, без Assist API, промпт про рецепты и готовку
- **Агент "Аналитик"** — модель DeepSeek Reasoner, история состояний включена, промпт про анализ данных

### Изменение агента

На карточке интеграции у каждого агента есть кнопка **Изменить** (Reconfigure) для редактирования параметров.

## Управление устройствами (Assist API)

Assist API позволяет LLM управлять устройствами Home Assistant через tool calling.

### Включение

1. Откройте настройки агента
2. В поле **Assist API** выберите один или несколько API:
   - **Assist** — стандартный API HA для управления устройствами (свет, розетки, замки, климат и т.д.)
   - Другие API (если установлены MCP-серверы или кастомные llm.API)
3. Сохраните

### Как это работает

1. Пользователь: *"Включи свет на кухне"*
2. LLM анализирует запрос и вызывает инструмент `HassTurnOn` с параметром `entity_id: light.kitchen`
3. Home Assistant выполняет действие
4. LLM формирует подтверждение: *"Свет на кухне включен"*

### Поддерживаемые действия

Через Assist API доступны все стандартные действия HA:
- Включение/выключение устройств
- Изменение яркости, цвета
- Управление климатом (температура, режим)
- Открытие/закрытие замков, штор
- Запуск скриптов и сцен
- И любые другие зарегистрированные сервисы

> **Внимание:** LLM может выполнять **любые** действия, доступные через выбранный API. Убедитесь, что это соответствует вашим требованиям безопасности.

## Инструмент истории состояний

Позволяет LLM запрашивать историю изменений состояний устройств для анализа трендов и событий.

### Включение

1. Откройте настройки агента
2. Включите **Инструмент истории состояний** (State History Tool)
3. Сохраните

### Как это работает

LLM получает инструмент `get_state_history`, который принимает:
- `entity_id` — идентификатор сущности (например, `sensor.temperature`)
- `hours` — глубина запроса (максимум 24 часа, по умолчанию 1 час)

### Примеры запросов

- *"Какая была температура в спальне за последние 6 часов?"*
- *"Сколько раз открывали входную дверь сегодня?"*
- *"Когда последний раз срабатывал датчик движения в коридоре?"*

LLM получит список изменений состояний с метками времени (до 20 последних записей) и сформирует ответ на основе этих данных.

### Требования

- Компонент **Recorder** должен быть настроен в HA (включен по умолчанию)
- Данные доступны за период, хранимый Recorder

## Мульти-агент (делегирование)

Когда у провайдера создано несколько агентов, каждый из них автоматически получает инструмент `ask_agent` для делегирования задач другим агентам.

### Как это работает

1. Создайте несколько агентов с разными специализациями
2. Каждый агент видит список «соседних» агентов
3. При получении вопроса не по своей теме, агент может вызвать `ask_agent` и передать запрос специалисту
4. Ответ специалиста интегрируется в общий ответ

### Пример

Допустим, у вас два агента на OpenAI:
- **"Домашний ассистент"** — управление устройствами, Assist API включен
- **"Аналитик энергии"** — история состояний включена, промпт про анализ потребления

Пользователь спрашивает "Домашнего ассистента": *"Сколько электроэнергии потрачено за сегодня?"*

Агент понимает, что это вопрос для аналитика, вызывает:
```
ask_agent(agent_name="Аналитик энергии", message="Сколько электроэнергии потрачено за сегодня?")
```

"Аналитик энергии" обрабатывает запрос и возвращает ответ, который "Домашний ассистент" передает пользователю.

> **Примечание:** Делегирование работает только между агентами **одного провайдера** (одной интеграции).

## Подключение MCP-серверов

SmartChain поддерживает [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) через встроенную интеграцию HA.

### Настройка

1. Установите интеграцию **Model Context Protocol** в HA:
   **Настройки > Устройства и службы > Добавить интеграцию > Model Context Protocol**
2. Укажите URL вашего MCP-сервера и авторизацию
3. После подключения MCP-сервер появится в списке доступных API
4. В настройках агента SmartChain выберите MCP API в поле **Assist API** (можно выбрать несколько)

### Что это дает

MCP-серверы предоставляют дополнительные инструменты для LLM. Например:
- Поиск в интернете
- Работа с файловой системой
- Запросы к базам данных
- Интеграция с внешними сервисами

Все инструменты MCP-сервера автоматически доступны LLM через tool calling.

## Распознавание изображений (Vision)

SmartChain поддерживает мультимодальные запросы — LLM может анализировать изображения с камер.

### Поддерживаемые провайдеры

Vision работает с моделями, поддерживающими изображения:
- **OpenAI** — GPT-4o, GPT-4.1
- **GigaChat** — GigaChat 2.0 (Max, Pro)
- **Ollama** — модели с vision (LLaVA, Gemma 3 и др.)
- **Anthropic** — Claude Sonnet, Opus

### Как это работает

1. В HA Voice Pipeline или через интеграцию отправьте изображение вместе с текстовым запросом
2. SmartChain конвертирует изображение в base64 и отправляет в LLM как мультимодальное сообщение
3. LLM анализирует изображение и отвечает

Большие изображения (>512 КБ) автоматически сжимаются через TurboJPEG (если установлен).

### Примеры

- *"Что ты видишь на камере у входной двери?"*
- *"Есть ли кто-то на парковке?"*
- *"Какая погода за окном по камере?"*

## Система навыков (Skills)

Навыки — это YAML-файлы с дополнительными знаниями и инструкциями, которые добавляются в системный промпт.

### Создание навыка

1. Создайте директорию `config/smartchain/skills/` (в корне конфигурации HA)
2. Создайте YAML-файл, например `weather_expert.yaml`:

```yaml
name: Эксперт по погоде
description: Знания о погоде и климате
prompt: |
  Ты хорошо разбираешься в погоде и климате.
  При вопросах о погоде используй данные датчиков температуры и влажности.
  Учитывай сезон и время суток при ответах.
  Можешь давать рекомендации по одежде на основе погоды.
```

3. Перезагрузите интеграцию SmartChain (или перезапустите HA)

### Формат файла навыка

```yaml
name: Название навыка          # обязательное
description: Описание навыка   # опциональное
prompt: |                      # обязательное
  Текст промпта, который будет
  добавлен к системному промпту.
```

### Примеры навыков

**Переводчик** (`translator.yaml`):
```yaml
name: Переводчик
description: Перевод текста между языками
prompt: |
  Если пользователь просит перевести текст,
  переведи его точно и естественно.
  Поддерживай русский, английский, немецкий, французский.
```

**Рецепты** (`recipes.yaml`):
```yaml
name: Домашний повар
description: Рецепты и советы по готовке
prompt: |
  Ты опытный повар. При вопросах о еде:
  - Предлагай рецепты с учетом имеющихся продуктов
  - Указывай время приготовления
  - Давай пошаговые инструкции
```

### Как это работает

Все `.yaml` файлы из `config/smartchain/skills/` загружаются при первом обращении к агенту. Промпты навыков добавляются в конец системного промпта каждого агента.

Навыки кэшируются — для применения изменений перезагрузите интеграцию.

## Сервис smartchain.ask

Сервис `smartchain.ask` позволяет отправлять запросы к LLM из автоматизаций, скриптов и интеграций.

### Параметры

| Параметр | Тип | Обязательный | Описание |
|----------|-----|:---:|----------|
| `message` | string | да | Текст сообщения для LLM |
| `entity_id` | string | нет | ID сущности агента. Если не указан — используется первый доступный |

### Вызов из автоматизаций

```yaml
service: smartchain.ask
data:
  message: "Какая сейчас температура в доме?"
response_variable: llm_response
```

Ответ доступен в переменной `llm_response.response`.

### Пример: Telegram-бот

Автоматизация, которая отвечает на сообщения Telegram через SmartChain:

```yaml
automation:
  - alias: "SmartChain Telegram Bot"
    trigger:
      - platform: event
        event_type: telegram_text
    action:
      - service: smartchain.ask
        data:
          message: "{{ trigger.event.data.text }}"
        response_variable: llm_response
      - service: telegram_bot.send_message
        data:
          target: "{{ trigger.event.data.chat_id }}"
          message: "{{ llm_response.response }}"
```

### Пример: Утренний брифинг

```yaml
automation:
  - alias: "Утренний брифинг"
    trigger:
      - platform: time
        at: "07:00:00"
    action:
      - service: smartchain.ask
        data:
          message: >
            Составь краткий утренний брифинг: погода, температура в доме,
            какие устройства включены, есть ли оповещения.
        response_variable: briefing
      - service: notify.mobile_app
        data:
          message: "{{ briefing.response }}"
```

### Пример: С конкретным агентом

```yaml
service: smartchain.ask
data:
  message: "Проанализируй потребление электроэнергии за сутки"
  entity_id: conversation.smartchain_analyst
response_variable: analysis
```

## Сервис smartchain.analyze_image

Сервис `smartchain.analyze_image` делает снимок с камеры HA, отправляет его мультимодальному LLM и возвращает текстовый анализ.

### Параметры

| Параметр | Тип | Обязательный | Описание |
|----------|-----|:---:|----------|
| `camera_entity_id` | string | да | Entity ID камеры (например, `camera.front_door`) |
| `prompt` | string | нет | Вопрос к LLM об изображении. По умолчанию: «Что ты видишь на этом изображении?» |
| `entity_id` | string | нет | ID агента SmartChain. Если не указан — используется первый доступный |

### Требования

Используйте мультимодальную модель, поддерживающую изображения (GPT-4o, GigaChat-Max, Claude Sonnet, Ollama с vision и др.).

### Пример: Анализ камеры при срабатывании датчика движения

```yaml
automation:
  - alias: "Анализ камеры при движении"
    trigger:
      - platform: state
        entity_id: binary_sensor.front_door_motion
        to: "on"
    action:
      - service: smartchain.analyze_image
        data:
          camera_entity_id: camera.front_door
          prompt: "Опиши, что происходит. Есть ли посторонние лица или подозрительная активность?"
        response_variable: analysis
      - service: notify.mobile_app
        data:
          message: "Движение у входа: {{ analysis.response }}"
```

## Панель SmartChain AI

SmartChain устанавливает дополнительную боковую панель **SmartChain AI**, доступную прямо в интерфейсе Home Assistant.

### Доступ к панели

После установки интеграции нажмите **SmartChain AI** в левой боковой панели Home Assistant.

### Вкладка «Генерация»

Позволяет создавать и редактировать автоматизации, скрипты, сцены и blueprint из текстового описания.

**Тип генерации** — выберите, что хотите создать:
- **Automation** — автоматизация (с триггерами, условиями и действиями)
- **Script** — скрипт (набор действий без триггеров)
- **Scene** — сцена (снимок состояний устройств)
- **Blueprint** — шаблон автоматизации для повторного использования

**Источник YAML** — две вкладки выбора исходного материала:
- **Вставить YAML** — вставьте существующий YAML вручную для редактирования
- **Выбрать из HA** — откройте список существующих автоматизаций/скриптов/сцен/blueprint из вашего HA и загрузите один из них в редактор

**Описание** — опишите задачу на естественном языке. Используйте кнопку выбора сущности (пикер) для вставки точных entity ID прямо из HA.

**Агент** — выберите, какой SmartChain-агент будет выполнять генерацию.

**Кнопка «Сгенерировать»** — LLM создаёт YAML на основе описания и контекста вашего HA (реальные entity ID, зоны, устройства).

**YAML-редактор** — сгенерированный YAML отображается в редакторе с подсветкой синтаксиса и нумерацией строк. Его можно отредактировать вручную.

**Валидация** — перед деплоем YAML автоматически проверяется: структура (наличие триггеров/действий), существование entity ID в HA, доступность упомянутых сервисов. Ошибки отображаются до деплоя.

**Кнопка «Задеплоить»** — отправляет YAML в Home Assistant. Автоматизации записываются в `automations.yaml`, скрипты — в `scripts.yaml`, сцены — в `scenes.yaml`, blueprint — в директорию `blueprints/`.

> **Предупреждение о blueprint:** Если загруженная автоматизация основана на blueprint, панель покажет предупреждение с рекомендацией редактировать исходный blueprint, а не саму автоматизацию.

#### Режим редактирования существующего YAML

Если в поле «Вставить YAML» или через «Выбрать из HA» загружен исходный YAML, генерация переходит в режим **улучшения**: LLM получает исходный YAML и описание желаемых изменений, после чего возвращает модифицированную версию, сохраняя существующую структуру.

### Вкладка «Анализ камеры»

Позволяет анализировать изображение с любой камеры HA с помощью мультимодального LLM.

1. Выберите камеру из списка
2. Введите вопрос (например, *"Есть ли кто-то у двери?"*)
3. Нажмите **Анализировать**
4. Результат отобразится под камерой

## Сервис smartchain.generate_automation

Программный интерфейс к генерации YAML — для использования из автоматизаций и скриптов.

### Параметры

| Параметр | Тип | Обязательный | Описание |
|----------|-----|:---:|----------|
| `description` | string | да | Описание на естественном языке |
| `yaml_type` | string | нет | Тип YAML: `automation`, `script`, `scene`, `blueprint`. По умолчанию: `automation` |
| `source_yaml` | string | нет | Исходный YAML для улучшения. Если указан — LLM модифицирует его, а не генерирует с нуля |
| `entity_id` | string | нет | ID агента SmartChain. Если не указан — используется первый доступный |

### Пример: Генерация автоматизации

```yaml
service: smartchain.generate_automation
data:
  description: >
    Когда открывается входная дверь после 23:00,
    включить свет в прихожей на 5 минут.
response_variable: result
```

Ответ содержит поле `yaml` с готовым YAML:

```yaml
# result.yaml — готовый YAML для вставки или деплоя
```

### Пример: Улучшение существующей автоматизации

```yaml
service: smartchain.generate_automation
data:
  description: "Добавь условие — только если никого нет дома"
  source_yaml: "{{ existing_yaml }}"
response_variable: result
```

## Сервис smartchain.deploy_automation

Деплой YAML в конфигурацию Home Assistant с автоматической перезагрузкой.

### Параметры

| Параметр | Тип | Обязательный | Описание |
|----------|-----|:---:|----------|
| `yaml` | string | да | YAML-текст для деплоя |
| `yaml_type` | string | нет | Тип YAML: `automation`, `script`, `scene`, `blueprint`. По умолчанию: `automation` |

### Маршрутизация по типу

| Тип | Куда записывается |
|-----|------------------|
| `automation` | `config/automations.yaml` |
| `script` | `config/scripts.yaml` |
| `scene` | `config/scenes.yaml` |
| `blueprint` | `config/blueprints/smartchain/` |

После записи файла соответствующий компонент HA перезагружается автоматически (`automation.reload`, `script.reload`, `scene.reload`).

### Пример: Генерация и деплой в одном потоке

```yaml
- service: smartchain.generate_automation
  data:
    description: "Выключать все огни в полночь"
  response_variable: generated
- service: smartchain.deploy_automation
  data:
    yaml: "{{ generated.yaml }}"
```

## Сервисы list_yaml и get_yaml

Сервисы для просмотра и загрузки существующих элементов HA — используются панелью SmartChain AI и доступны из автоматизаций.

### smartchain.list_yaml

Возвращает список существующих автоматизаций, скриптов, сцен или blueprint.

**Параметры:**

| Параметр | Тип | Обязательный | Описание |
|----------|-----|:---:|----------|
| `yaml_type` | string | да | Тип: `automation`, `script`, `scene`, `blueprint` |

**Пример:**

```yaml
service: smartchain.list_yaml
data:
  yaml_type: automation
response_variable: items
```

Ответ содержит поле `items` — список объектов с полями `id` и `alias`.

### smartchain.get_yaml

Возвращает YAML-исходник конкретного элемента по его ID.

**Параметры:**

| Параметр | Тип | Обязательный | Описание |
|----------|-----|:---:|----------|
| `yaml_type` | string | да | Тип: `automation`, `script`, `scene`, `blueprint` |
| `item_id` | string | да | ID элемента (из ответа `list_yaml`) |

**Пример:**

```yaml
service: smartchain.get_yaml
data:
  yaml_type: automation
  item_id: "1234567890abcdef"
response_variable: item
```

Ответ содержит поле `yaml` с YAML-текстом.

## AI Task (автоматизации)

SmartChain предоставляет сущность **AI Task** для генерации структурированных данных в автоматизациях.

### Использование

```yaml
service: ai_task.generate_data
data:
  task_name: "Рекомендация по климату"
  entity_id: ai_task.smartchain_gigachat
  instructions: "Предложи оптимальную температуру на основе текущих данных"
  structure:
    - name: temperature
      type: number
      description: "Рекомендуемая температура"
    - name: reason
      type: string
      description: "Обоснование"
response_variable: result
```

Ответ в `result` будет содержать структурированные данные (JSON).

### Возможности

- Генерация текста
- Структурированный вывод (JSON по заданной схеме)
- Tool calling (те же инструменты, что и у conversation agent)

> **Требование:** Home Assistant 2025.3+ для поддержки AI Task.

## Системный промпт (Jinja2)

Системный промпт определяет поведение ассистента. Он поддерживает Jinja2-шаблоны с доступом к данным HA.

### Промпт по умолчанию

```
You are a smart home voice assistant {{ ha_name }} powered by Home Assistant.
Answer truthfully and to the point. Answer in plain text, briefly and clearly.
Answer in the user's language.
```

### Доступные переменные

| Переменная | Описание |
|-----------|----------|
| `{{ ha_name }}` | Название дома (из настроек HA) |

### Пример кастомного промпта

```
Ты — умный ассистент для дома «{{ ha_name }}».
Отвечай кратко и по делу, на языке пользователя.
Используй данные датчиков для точных ответов.
Если не знаешь ответ — скажи об этом.
При управлении устройствами всегда подтверждай выполненное действие.
```

### Автоматический контекст устройств

Когда **Assist API не подключен**, SmartChain автоматически добавляет к промпту список всех комнат, устройств и их текущих состояний. Это позволяет LLM отвечать на вопросы о состоянии дома даже без tool calling.

Когда **Assist API подключен**, контекст устройств предоставляется самим Assist API.

## FAQ и решение проблем

### Интеграция не найдена после установки
Убедитесь, что папка `smartchain` находится в `config/custom_components/smartchain/` и перезапустите HA.

### Ошибка "Cannot connect"
- Проверьте API-ключ
- Для Ollama — убедитесь, что сервер запущен и доступен по указанному адресу
- Для облачных провайдеров — проверьте наличие интернета

### LLM не управляет устройствами
- Убедитесь, что **Assist API** выбран в настройках агента
- Проверьте, что устройства доступны в HA и не отключены

### История состояний не работает
- Включите **Инструмент истории состояний** в настройках агента
- Убедитесь, что компонент **Recorder** настроен в HA

### Навыки не загружаются
- Проверьте путь: `config/smartchain/skills/*.yaml`
- Убедитесь, что файлы содержат обязательные поля `name` и `prompt`
- Перезагрузите интеграцию после добавления навыков

### Панель SmartChain AI не отображается
- Перезапустите Home Assistant после установки интеграции
- Проверьте, что в логах HA нет ошибок при регистрации панели

### analyze_image возвращает ошибку
- Убедитесь, что используемая модель поддерживает vision (GPT-4o, GigaChat-Max, Claude Sonnet и т.д.)
- Для GigaChat — модель должна быть GigaChat-Max или GigaChat-Pro

### Сгенерированный YAML не проходит валидацию
- Проверьте, что entity ID из YAML действительно существуют в вашем HA
- Уточните описание — укажите конкретные имена устройств или используйте пикер сущностей
- Попробуйте другой агент с более мощной моделью (GPT-4.1, Claude Sonnet)

### Как экономить токены?
- Используйте короткий системный промпт
- Отключите историю сообщений, если она не нужна
- Выбирайте модели `mini`/`nano`/`lite` для простых задач
- Кэширование промптов работает автоматически (30 секунд TTL)
- Для локальных задач используйте Ollama (бесплатно)

### Как использовать с Telegram?
Настройте автоматизацию с сервисом `smartchain.ask` — см. [пример выше](#пример-telegram-бот).

### Поддержка нескольких языков
SmartChain поддерживает все языки, поддерживаемые Home Assistant. Язык ответа определяется LLM — укажите в системном промпте: *"Отвечай на языке пользователя"*.
